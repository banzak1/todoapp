## 2. Ambiente de Desenvolvimento

**Decisão:** Usar WSL2 Ubuntu como ambiente principal
- Docker roda nativamente no Linux
- Kafka, Redis, Terraform são cidadãos de primeira classe no Unix
- CI/CD pipelines rodam Linux em 95% dos casos
- VS Code + extensão Remote-WSL para editar no Windows

**Setup:**
```
Windows (host) → VS Code + Remote-WSL
WSL2 Ubuntu → código, Docker, CLI tools, Claude Code
```

## 3. Arquitetura do Projeto — 8 Fases

O **todoApp** não é um simples app de tarefas — é um laboratório de experimentação progressiva. A cada fase, uma nova camada é adicionada **ao mesmo app**, simulando evolução real de sistemas.

| Fase | O que construir | Tecnologias praticadas |
|------|----------------|------------------------|
| **1. Fundação** | API REST Java/Spring Boot + PostgreSQL | Java 21, Spring Boot 3, JPA, JUnit 5, Mockito |
| **2. Containerização** | Docker + docker-compose | Docker, redes, volumes, multi-stage builds |
| **3. Mensageria** | Kafka para eventos (notificações, audit log) | Kafka, event-driven architecture |
| **4. Cache** | Redis para caching e rate limiting | Redis, estratégias de cache |
| **5. Observabilidade** | Grafana + Prometheus, logs estruturados | Métricas, dashboards, alertas |
| **6. Infra como Código** | Terraform para provisionar recursos | Terraform, cloud basics |
| **7. CI/CD** | Pipeline no GitHub Actions | CI/CD, testes automatizados, deploy |
| **8. Escalabilidade** | Kubernetes (minikube/k3s) | K8s, horizontal scaling, health checks |
